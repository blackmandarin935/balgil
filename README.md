# 밝길 (Balgil)

> 가장 빠른 길 말고, 밤에 가장 안전한 길

밤길 보행 안전 사각지대 문제를 해결하기 위한 웹 서비스입니다. 실제 도로망 위에서 범죄예방 환경설계(CPTED) 5원리를 지표로 계산해, 어두운 골목 대신 안전한 경로를 안내합니다.

## 언어와 사용성

- 한국어, English, 日本語, 简体中文, Tiếng Việt, Español, Français를 화면에서 바로 전환할 수 있으며 선택한 언어는 이 기기에 기억됩니다.
- 주소 검색은 Cloudflare Function을 통해 Nominatim에 요청하며, 응답이 어려울 때만 브라우저에서 직접 검색합니다.
- CSV는 UTF-8·EUC-KR 인코딩과 큰따옴표가 포함된 열을 처리합니다.
- 위험 신고는 **이 기기의 브라우저에만** 보관되어, 같은 지역을 다시 불러왔을 때 지도에 표시됩니다. 기관에 자동 접수되지는 않습니다.

**[▶ 바로 써보기](https://01109c25.balgil.pages.dev)**

## 왜 만들었나

학원을 마치고 집에 오는 길은 늘 밤입니다. 지도 앱이 알려주는 최단 경로는 가로등이 드문 골목길이라, 저는 매번 큰길로 돌아옵니다. 3분 더 걸리지만 마음이 편합니다.

문제는 이겁니다. **저는 동네를 아니까 돌아갈 수 있지만, 처음 온 사람에게 지도 앱은 가장 어두운 길을 알려줍니다.**

기존 지도 앱이 경로를 고르는 기준에는 "밤"이라는 조건이 없습니다.

```
거리 최소 · 시간 최소 · 요금 최소
```

밝길은 여기에 **안전**을 넣었습니다.

---

## 개발 과정에서 만난 벽

처음에는 단순했습니다. "가로등이 많으면 안전하다."

그런데 OpenStreetMap에서 대전 둔산동 가로등 데이터를 가져오니 **0개**였습니다. 실제로는 수백 개가 있는 동네인데 지도에는 등록되어 있지 않았습니다.

여기서 질문을 바꿨습니다.

> ~~가로등 데이터를 어떻게 구할 것인가~~
> **가로등이 없어도 밤길 위험도를 잴 수 있는가**

답은 CPTED(범죄예방 환경설계)에 있었습니다. 밤길 안전은 밝기만으로 결정되지 않습니다. **창문이 많은 골목은 가로등이 어두워도 상대적으로 안전합니다.** 사람의 시선이 닿기 때문입니다. 반대로 가로등만 밝은 공터는 위험합니다.

그리고 건물 데이터는 한국 OSM에 아주 촘촘하게 등록되어 있습니다.

---

## CPTED 5지표

이론을 지도에서 잴 수 있는 값으로 변환했습니다.

| 지표 | CPTED 원리 | 측정 방법 (OSM 태그) |
|---|---|---|
| 조도 | 조명 | `highway=street_lamp`, `lit=yes/no` |
| 주변 시선 | 자연적 감시 | 반경 70m 건물 수 `building`, CCTV `man_made=surveillance` |
| 밤에 사람 | 활용성 증대 | 반경 120m 상점·음식점·정류장 |
| 길 모양 | 접근통제 | 도로 등급 `highway`, 막다른 길, 보도 `sidewalk` |
| 대피할 곳 | 영역성 | 24시 편의점 `shop=convenience`, 파출소 `amenity=police` |

대피 지표는 한국 상황을 반영한 것입니다. 24시간 편의점은 **여성안심지킴이집**으로 지정되어 경찰 112 핫라인이 연결된 공식 대피소 역할을 합니다.

## 핵심 원리

### 조도 — 거리 역제곱 법칙

빛의 세기는 거리의 제곱에 반비례하고, 여러 가로등의 빛은 더해집니다.

```
E(p) = Σ  k / (d² + r₀²)
```

```javascript
let E = 0;
for (const j of D.lamps.near(la, lo)) {
  const p = D.lamps.pts[j];
  const d = dist(la, lo, p.lat, p.lon);
  E += 9000 / (d * d + 800);
}
```

### 거리 — 하버사인 공식

지구는 평면이 아니므로 구면 위의 거리를 구합니다.

```
a = sin²(Δφ/2) + cos φ₁ · cos φ₂ · sin²(Δλ/2)
d = 2R · arcsin(√a)
```

### 경로 비용 — 거리의 정의를 바꾸기

이 프로젝트의 핵심 아이디어입니다.

```
경로비용 = 실제거리 × ( 1 + α · (1 − 안전점수) )
```

안전 경로를 찾는 새 알고리즘을 만든 게 아닙니다. **어두운 길이 실제보다 "멀게 느껴지도록" 거리를 재정의한 뒤, 평범한 다익스트라를 돌립니다.**

- `α = 0` → 최단 경로와 완전히 동일
- `α`가 클수록 → 돌아가더라도 안전한 길

## 주요 기능

- **실제 지도 기반** — 전국 어디서나, 별도 데이터 준비 없이
- **이중 경로 비교** — 안전 경로(초록)와 최단 경로(빨강 점선)를 동시에
- **상황별 프리셋** — 기본 / 혼자 늦은 밤 / 아이와 함께 / 인적 드문 길 피하기
- **AI 신고 접수** — 말로 적은 신고를 Gemini가 유형·심각도로 분류
- **결과를 말로 설명** — "조금 조심하세요 · 대피할 곳이 가깝고, 가로등이 거의 없어요"
- **상대 평가** — 절대 점수 대신 "이 동네 길 중 상위 34%"
- **데이터 보완** — 공공데이터포털 CSV 넣기, 지도에 직접 가로등 찍기

## 성능 설계

동네 하나에 도로 구간이 수천~수만 개입니다. 그냥 만들면 슬라이더를 움직일 때마다 화면이 멈춥니다.

**가장 중요한 것은 비싼 계산과 싼 계산을 분리한 것입니다.**

| 언제 | 무엇을 | 시간 |
|---|---|---|
| 데이터 불러올 때 (한 번) | 지표 5개 계산 → `Float32Array` 저장 | 11ms |
| 슬라이더 움직일 때 (매번) | 저장된 값에 곱셈만 | 0.3ms |
| 출발·도착 찍을 때 | 다익스트라 | 1.2ms |

*측정 환경: 개발 중 합성 데이터 기준 (노드 4,900 · 구간 9,660)*

그 외 적용한 것:

- **격자 공간 색인** — "반경 70m에 건물 몇 개?"를 O(N) 대신 주변 9칸만 확인
- **요청 병렬 처리** — 도로와 시설 데이터를 `Promise.all`로 동시에
- **CSR 그래프 구조** — 인접 리스트를 정수 배열 3개(`head`, `adjTo`, `adjE`)로
- **레이어 묶기** — 도로 1만 개를 색 8단계로 묶어 레이어 8개만 생성
- **캔버스 렌더링** — Leaflet `preferCanvas: true`

```
balgil/
├── index.html                  앱 본체
│   ├── Grid()                  공간 색인

│   ├── Heap()                  우선순위 큐
│   └── findPath()              다익스트라
│
├── functions/
│   └── api/
│       ├── analyze.js          AI 신고 분석 (Gemini)
│       └── geocode.js          주소 검색 (Nominatim 프록시)
│
```

`functions/` 폴더의 경로가 곧 API 주소가 됩니다. 별도 설정 파일이 없습니다.

```
functions/api/analyze.js  →  /api/analyze
functions/api/geocode.js  →  /api/geocode
```

## API 키는 왜 서버에 두었나

브라우저 코드에 API 키를 넣으면 F12만 눌러도 누구나 볼 수 있습니다. 그래서 중간에 서버를 하나 두었습니다.

```
[브라우저]  →  [Cloudflare Functions]  →  [Gemini API]
                  ↑ 키는 여기에만
```

예상하지 못한 장점이 있었습니다. **AI를 바꿔도 브라우저 코드를 고칠 필요가 없습니다.** 개발 중 Claude API에서 Gemini API로 교체했는데, 서버 파일 하나만 수정하면 끝이었습니다.

## 직접 돌려보기

### 로컬에서

```bash
git clone https://github.com/blackmandarin935/balgil.git
cd balgil
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속.

> **주의**: `index.html`을 더블클릭해서 여는 방식(`file://`)은 작동하지 않습니다. 브라우저 보안 정책 때문에 외부 요청이 모두 차단됩니다. 반드시 로컬 서버를 띄우거나 배포된 주소로 접속하세요.

로컬에서는 `/api/analyze`가 없으므로 AI 분석 대신 규칙 기반 분류가 작동합니다. 화면에 `엔진 — 규칙 분석`이라고 표시됩니다.

### 배포하기

1. 이 저장소를 fork
2. [Cloudflare Dashboard](https://dash.cloudflare.com) → Workers & Pages → Create → Pages → **Connect to Git**
3. 빌드 설정
   | 항목 | 값 |
   |---|---|
   | Framework preset | None |
   | Build command | *(비워두기)* |
   | Build output directory | *(비워두기)* |
4. Settings → Variables and Secrets → **Secret** 타입으로 추가
   ```
   GEMINI_API_KEY = (aistudio.google.com에서 발급)
   ```
5. Deployments → **Retry deployment** (환경 변수는 재배포해야 반영됩니다)

> 대시보드의 드래그 업로드 방식은 Functions를 컴파일하지 않습니다. 반드시 Git 연동을 사용하세요.

### 잘 되고 있는지 확인

배포 후 `/api/analyze` 주소를 브라우저에서 열면 상태가 JSON으로 나옵니다.

```
https://your-project.pages.dev/api/analyze
```

| 응답 | 의미 |
|---|---|
| `"상태": "정상"` + 모델 목록 | 정상 |
| `"상태": "키 거부됨"` | API 키가 잘못됨 |
| `"상태": "키 없음"` | 재배포가 필요함 |

## 알려진 한계

- **가중치가 검증되지 않았습니다.** 안전 점수의 가중치(0.30 / 0.25 / 0.15 / 0.20 / 0.10)는 CPTED 이론을 참고해 정한 값이지, 실제 범죄 통계로 검증한 값이 아닙니다.
- **가로등 데이터가 부족합니다.** 한국 OSM에는 가로등이 거의 등록되어 있지 않아, 다지표 방식으로 보완했습니다. 가로등이 실제로 많은 동네와 적은 동네를 구분하지 못합니다.
- **신고는 이 기기에만 저장됩니다.** 같은 기기에서는 다시 불러올 수 있지만, 여러 사람이 공유하거나 기관에 접수하려면 데이터베이스와 별도 접수 연동이 필요합니다.
- **공용 서버에 의존합니다.** Overpass API가 붐비면 느려집니다. 서버 4곳 자동 전환으로 대응했지만 근본적 해결은 아닙니다.
- **지하보도·육교·아파트 단지 내부**는 데이터가 부족하거나 별도 처리가 필요합니다.

## 앞으로 하고 싶은 것

- [ ] **현장 조사** — 동네를 직접 걸으며 가로등 위치 기록, 수집한 데이터를 OpenStreetMap에 기여
- [ ] **사진 AI 판정** — 밤길 사진을 찍으면 AI가 조도와 시야를 판정
- [ ] **지자체 제안서 자동 생성** — 쌓인 신고를 공문 형식의 개선 요청서로 변환
- [ ] **시간대 반영** — 상점 영업시간(`opening_hours`)으로 밤 9시와 새벽 2시를 구분
- [ ] **교통약자 모드** — 턱 높이(`kerb`), 경사(`incline`), 계단(`steps`) 기반 경로

세 번째가 가장 중요합니다. 개인이 안전한 길로 다니는 것은 위험을 **피하는** 것이고, 위험한 길 자체를 고쳐야 **해결**입니다.
